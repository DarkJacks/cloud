# ☁️ CloudStorage - Облачное хранилище файлов

[![Node.js Version](https://img.shields.io/badge/Node.js-18.x-green.svg)](https://nodejs.org/)
[![MongoDB](https://img.shields.io/badge/MongoDB-6.x-brightgreen.svg)](https://www.mongodb.com/)
[![License](https://img.shields.io/badge/License-MIT-blue.svg)](LICENSE)
[![Version](https://img.shields.io/badge/Version-2.0.1-orange.svg)]()

**CloudStorage** - это современное, безопасное и функциональное облачное хранилище файлов с веб-интерфейсом, админ-панелью и REST API. Проект позволяет пользователям загружать, хранить и управлять файлами, а также предоставляет администраторам полный контроль над системой.

## ✨ Возможности

### 👤 Для пользователей
- 📁 **Управление файлами** - создание папок, загрузка, переименование, удаление файлов
- 🖼️ **Предпросмотр файлов** - просмотр изображений, видео, PDF в браузере
- 🔗 **Общие ссылки** - создание публичных и приватных ссылок для скачивания
- 📊 **Статистика хранилища** - отслеживание использованного места с визуальным прогресс-баром
- 🔐 **Безопасная авторизация** - JWT токены, хеширование паролей
- 🎫 **Управление токенами** - просмотр и удаление своих access tokens
- 🎨 **Темная и светлая тема** - выбор комфортного режима с сохранением в localStorage
- 📱 **Адаптивный дизайн** - поддержка всех типов устройств

### 👑 Для администраторов и модераторов
- 📊 **Дашборд** - полная статистика системы
- 👥 **Управление пользователями** - блокировка/разблокировка, изменение ролей, удаление
- 📁 **Модерация файлов** - просмотр и удаление любых файлов пользователей
- 🔑 **Управление API ключами** - создание и удаление API ключей
- 🎫 **Токены регистрации** - создание одноразовых токенов для безопасной регистрации
- 🔐 **Access Tokens** - управление токенами доступа пользователей
- ⚙️ **Настройка системы** - изменение названия хранилища

### 🌍 Публичные функции (без регистрации)
- 📤 **Загрузка файлов** - быстрая загрузка с выбором срока хранения
- 🔗 **Прямые ссылки** - получение MD5 ссылки на файл
- 🖼️ **Просмотр изображений** - открытие изображений прямо в браузере
- ⏰ **Выбор срока хранения** - 1, 7, 30, 90, 180 дней или навсегда

### 🔌 API Интерфейс
- 📝 **Регистрация по токену** - безопасная регистрация (рекомендуемый способ)
- 🔑 **Регистрация по API ключу** - legacy метод
- 🔐 **Вход по Access Token** - аутентификация через API
- 📁 **Управление файлами** - получение списка, удаление через API
- 👤 **Информация о пользователе** - получение данных о текущем пользователе
- 🎫 **Управление токенами** - просмотр и удаление своих токенов

## 🛠️ Технологии

### Backend
- **Node.js** - среда выполнения JavaScript
- **Express.js** - веб-фреймворк
- **MongoDB** - база данных
- **Mongoose** - ODM для MongoDB
- **JWT** - аутентификация
- **bcrypt** - хеширование паролей
- **Multer** - загрузка файлов
- **Helmet** - защита HTTP заголовков
- **Compression** - сжатие ответов
- **Morgan** - логирование (только в файл)

### Frontend
- **HTML5** - структура страниц
- **CSS3** - стилизация, анимации, адаптивность
- **JavaScript (ES6+)** - интерактивность
- **Fetch API** - работа с сервером
- **LocalStorage** - сохранение настроек

## 🚀 Быстрая установка

### Способ 1: Автоматическая установка (рекомендуемый)

1. **Создайте папку проекта**:
```bash
mkdir cloud-storage
cd cloud-storage
```

2. **Создайте файл `create-complete-project.js`** со скриптом установки (см. ниже)

3. **Запустите скрипт**:
```bash
node create-complete-project.js
```

4. **Установите зависимости**:
```bash
npm install
```

5. **Запустите MongoDB** (в отдельном терминале):
```bash
mongod
```

6. **Запустите сервер**:
```bash
npm start
```

7. **Откройте браузер**: http://localhost:3000

### Способ 2: Ручная установка

1. **Клонируйте репозиторий**:
```bash
git clone https://github.com/yourusername/cloud-storage.git
cd cloud-storage
```

2. **Установите зависимости**:
```bash
npm install
```

3. **Настройте MongoDB**:
```bash
# Убедитесь, что MongoDB запущена
mongod
```

4. **Запустите сервер**:
```bash
npm start
```

## 📚 API Документация

### Доступные версии

| Версия | Описание | URL |
|--------|----------|-----|
| **v1.0.0** | Базовая документация | /api-docs-v1.0.0.html |
| **v2.0.0** | Расширенная документация | /api-docs-v2.0.0.html |
| **v2.0.1** | С кнопками копирования | /api-docs-v2.0.1.html |

### Основные API эндпоинты

#### Аутентификация
```http
POST /api/register-with-token    - Регистрация по токену
POST /api/register                - Регистрация по API ключу (legacy)
POST /api/login                   - Вход с access token
GET  /api/me                      - Информация о пользователе
POST /api/create-registration-token - Создание токена (admin/moderator)
```

#### Работа с файлами
```http
GET    /api/files                 - Получение списка файлов
DELETE /api/files/:fileId         - Удаление файла
POST   /api/files/share/:fileId   - Создание общей ссылки
```

#### Пользователь
```http
GET    /api/my-tokens             - Список своих токенов
DELETE /api/tokens/:tokenId       - Удаление токена
```

#### Административные
```http
GET    /api/admin/dashboard       - Статистика
GET    /api/admin/users           - Список пользователей
PUT    /api/admin/users/:userId   - Обновление пользователя
DELETE /api/admin/users/:userId   - Удаление пользователя
GET    /api/admin/files           - Все файлы
POST   /api/admin/api-keys        - Создание API ключа
POST   /api/admin/registration-tokens - Создание токена регистрации
POST   /api/admin/access-tokens   - Создание access token
```

#### Публичные
```http
POST   /public/upload             - Загрузка файла
GET    /public/download/:md5      - Скачивание файла
GET    /public/shared/:token      - Скачивание по общей ссылке
```

## 🔒 Безопасность

- **Пароли** - хешируются с помощью bcrypt
- **JWT токены** - для аутентификации
- **CORS** - настройка для безопасных запросов
- **Rate limiting** - защита от DDoS
- **Helmet** - защита HTTP заголовков
- **Валидация** - проверка всех входных данных

- **CSRF защита** - токены для форм

## 📊 Технические детали

### Хранилище
- **Лимит пользователя**: 1 GB по умолчанию
- **Максимальный размер файла**: 100 MB
- **Публичные файлы**: Хранятся 7 дней (настраивается)
- **Поддерживаемые форматы**: Все типы файлов

### Производительность
- **Сжатие ответов** - compression
- **Кэширование** - миниатюр и статики
- **Индексы MongoDB** - оптимизация запросов
- **Rate limiting** - защита от перегрузок

### Логирование
- **Логи запросов** - сохраняются в `logs/access.log`
- **Логи ошибок** - выводятся в консоль
- **Логирование отключено** в консоли для чистоты вывода

## 🐛 Устранение неполадок

### Ошибка подключения к MongoDB
```bash
# Убедитесь, что MongoDB запущена
mongod

# Проверьте строку подключения в .env
MONGODB_URI=mongodb://localhost:27017/cloud_storage

# Проверьте, что база данных существует
mongo
use cloud_storage
```

### Ошибка 401 Unauthorized
```bash
# Проверьте токен в localStorage
# Очистите кэш браузера (Ctrl+F5)
# Войдите заново в систему
```

### Ошибка 500 Internal Server Error
```bash
# Проверьте логи в папке logs/
# Проверьте консоль сервера
# Убедитесь, что все модели созданы
```

### Проблемы с загрузкой файлов
```bash
# Проверьте права на папки uploads и public_uploads
chmod 755 server/uploads
chmod 755 server/public_uploads

# Убедитесь, что папки существуют
# Проверьте размер файла (макс. 100 MB)
```

### Ошибка "OverwriteModelError"
```bash
# Убедитесь, что модели используют проверку существования
module.exports = mongoose.models.ModelName || mongoose.model('ModelName', schema);
```

### Проблемы с темной темой
```bash
# Очистите localStorage
localStorage.removeItem('theme');

# Перезагрузите страницу
location.reload();
```
