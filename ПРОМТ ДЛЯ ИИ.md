# 🤖 ПОЛНЫЙ ПРОМТ ДЛЯ СОЗДАНИЯ ПРОЕКТА ОБЛАЧНОГО ХРАНИЛИЩА

## 📋 ПРОМТ ДЛЯ ИСКУССТВЕННОГО ИНТЕЛЛЕКТА

```
Создай полноценное веб-приложение облачного хранилища файлов с нуля. Проект должен быть полностью рабочим и готовым к развертыванию.

## 🛠️ ТЕХНОЛОГИЧЕСКИЙ СТЕК
- **Backend**: Node.js + Express.js
- **Database**: MongoDB + Mongoose ODM
- **Frontend**: HTML5, CSS3, Vanilla JavaScript (без фреймворков)
- **Аутентификация**: JWT токены
- **Загрузка файлов**: Multer
- **Безопасность**: bcrypt, helmet, compression, rate-limit

## 👤 ФУНКЦИОНАЛ ДЛЯ ПОЛЬЗОВАТЕЛЕЙ

### 1. Регистрация и авторизация
- Регистрация с username, email (опционально), password
- Вход в систему с JWT токеном
- Выход из системы
- Восстановление пароля (опционально)

### 2. Управление файлами
- Создание папок
- Загрузка файлов (до 100MB)
- Переименование файлов и папок
- Удаление файлов и папок (с рекурсивным удалением содержимого)
- Навигация по папкам (вход/выход)
- Хлебные крошки для навигации
- Отображение текущего пути

### 3. Предпросмотр файлов
- Миниатюры для изображений (jpg, png, gif, webp)
- Превью видео (первый кадр)
- Иконки для различных типов файлов:
  - 📑 PDF
  - 📘 Word
  - 📊 Excel
  - 📽️ PowerPoint
  - 🎬 Видео
  - 🎵 Аудио
  - 🗜️ Архивы
  - ⚙️ Исполняемые файлы
  - 📝 Текстовые файлы

### 4. Общие ссылки
- Создание публичных ссылок на файлы
- Приватные ссылки (только для владельца)
- Настройка срока действия ссылки (1-365 дней или бессрочно)
- Копирование ссылки в буфер обмена
- Статистика скачиваний

### 5. Информация о хранилище
- Отображение использованного места
- Визуальный прогресс-бар
- Цветовая индикация заполнения:
  - Зеленый: до 70%
  - Оранжевый: 70-90%
  - Красный: более 90%

### 6. Интерфейс пользователя
- Кнопки навигации: Назад, Вверх, Главная
- Сортировка файлов: по имени, дате, размеру
- Поиск файлов
- Адаптивный дизайн
- Темная и светлая тема (сохраняется в localStorage)
- Всплывающие уведомления (тосты) с автозакрытием через 3 секунды

## 🔌 API ИНТЕРФЕЙС

### Методы аутентификации
1. **API Key** (legacy) - для регистрации новых пользователей
2. **Registration Token** (рекомендуемый) - безопасная регистрация
3. **Access Token** - для доступа к защищенным эндпоинтам

### API Эндпоинты

#### Регистрация и авторизация
```
POST   /api/register-with-token     - Регистрация по токену
POST   /api/register                - Регистрация по API ключу (legacy)
POST   /api/login                   - Вход с access token
GET    /api/me                      - Информация о пользователе
POST   /api/create-registration-token - Создание токена (admin/moderator)
```

#### Работа с файлами
```
GET    /api/files                   - Получение списка файлов
DELETE /api/files/:fileId           - Удаление файла
POST   /api/files/share/:fileId     - Создание общей ссылки
```

#### Управление токенами
```
GET    /api/my-tokens               - Список своих токенов
DELETE /api/tokens/:tokenId         - Удаление токена
```

#### Административные
```
GET    /api/admin/dashboard         - Статистика
GET    /api/admin/users             - Список пользователей
PUT    /api/admin/users/:userId     - Обновление пользователя
DELETE /api/admin/users/:userId     - Удаление пользователя
GET    /api/admin/files             - Все файлы
POST   /api/admin/api-keys          - Создание API ключа
DELETE /api/admin/api-keys/:keyId   - Удаление API ключа
POST   /api/admin/registration-tokens - Создание токена регистрации
DELETE /api/admin/registration-tokens/:tokenId - Удаление токена
POST   /api/admin/access-tokens     - Создание access token
DELETE /api/admin/access-tokens/:tokenId - Удаление access token
```

#### Публичные (без авторизации)
```
GET    /public/upload               - Страница загрузки
POST   /public/upload               - Загрузка файла
GET    /public/download/:md5        - Скачивание файла
GET    /public/shared/:token        - Скачивание по общей ссылке
```

## 🗄️ МОДЕЛИ ДАННЫХ

### User
```javascript
{
    username: String (unique, required),
    email: String (unique, required),
    password: String (hashed),
    role: ['user', 'moderator', 'admin'],
    isActive: Boolean,
    storageUsed: Number,
    storageLimit: Number,
    createdAt: Date,
    lastActive: Date
}
```

### File
```javascript
{
    name: String,
    type: ['file', 'folder'],
    parentId: String,
    owner: ObjectId (ref User),
    path: String,
    size: Number,
    mimeType: String,
    createdAt: Date,
    updatedAt: Date
}
```

### ApiKey
```javascript
{
    key: String (unique),
    name: String,
    owner: ObjectId (ref User),
    permissions: [String],
    isActive: Boolean,
    expiresAt: Date,
    lastUsed: Date
}
```

### AccessToken
```javascript
{
    token: String (unique),
    userId: ObjectId (ref User),
    permissions: [String],
    isActive: Boolean,
    expiresAt: Date,
    lastUsed: Date
}
```

### RegistrationToken
```javascript
{
    token: String (unique),
    createdBy: ObjectId (ref User),
    maxUses: Number,
    usedCount: Number,
    isActive: Boolean,
    expiresAt: Date
}
```

### PublicFile
```javascript
{
    md5: String (unique),

    path: String,
    size: Number,
